    Create `scripts/cleanup_physiology_zones.py` following the verify_analytics.py structural pattern.

    Structure:
    - `load_dotenv()` at module level before any `from coach.` import
    - `from coach.coaching.fitness_test_processor import PLAUSIBLE_BOUNDS` — import, do not redefine
    - `logger = logging.getLogger(__name__)`
    - `def main()` with `argparse.ArgumentParser()` adding `--confirm` flag (store_true, default False)
    - `if __name__ == "__main__": main()`

    Logic in `main()`:
    1. Call `get_supabase()` to get the DB client
    2. Query all rows from `physiology_zones`: select `id`, `discipline`, `valid_from`, `method`, `ftp_w`, `threshold_pace_s_per_km`, `css_pace_s_per_100m`
    3. For each row, determine the relevant value and bounds:
       - `discipline == 'bike'`: check `ftp_w` against `PLAUSIBLE_BOUNDS['ftp_bike_20min']` → (80, 450) W
       - `discipline == 'run'`: check `threshold_pace_s_per_km` against `PLAUSIBLE_BOUNDS['threshold_run_30min']` → (150, 360) s/km
       - `discipline == 'swim'`: check `css_pace_s_per_100m` against `PLAUSIBLE_BOUNDS['css_swim_400_200']` → (70, 150) s/100m
       - If the relevant column is None → row is not out-of-bounds (only check rows with actual values)
    4. Collect out-of-bounds rows (list of dicts with id, discipline, valid_from, method, value, bounds)

    Dry-run output (no --confirm): per D-07
    ```
    === Cleanup physiology_zones (DRY RUN) ===
    Righe fuori bounds che verrebbero cancellate:

    [bike] id=<uuid>  ftp_w=850W  FUORI RANGE (80-450W)  (metodo: 20min_test, data: 2026-05-01)
    [run]  id=<uuid>  threshold=0.5 s/km  FUORI RANGE (150-360 s/km)  (metodo: 30min_test, data: 2026-05-10)

    Totale: 2 righe da cancellare (0 disciplina bike, 1 disciplina run, 1 disciplina swim)
    Nessuna modifica al DB. Rieseguire con --confirm per cancellare.
    ```
    If no out-of-bounds rows:
    ```
    Nessuna riga fuori bounds. DB già pulito.
    ```

    --confirm output: per D-08
    ```
    === Cleanup physiology_zones (CONFIRM) ===
    Cancellazione in corso...
    DELETED [bike] id=<uuid>  ftp_w=850W  (metodo: 20min_test, data: 2026-05-01)
    DELETED [run]  id=<uuid>  threshold=0.5 s/km  (metodo: 30min_test, data: 2026-05-10)

    Cancellate: 2 righe totali (bike: 1, run: 1, swim: 0)
    ```

    Delete by `id` (UUID): `sb.table("physiology_zones").delete().eq("id", row_id).execute()`
    Log each deletion with `logger.info` before the delete call.

    Wrap in try/except at the loop level — one failed delete should not abort remaining rows. Log the exception with `logger.exception`.
