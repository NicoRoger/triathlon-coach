    Logic in `main()`:
    1. Define the extended June 2026 window: `cutoff_from = "2026-06-01T00:00:00+00:00"`, `cutoff_to = "2026-06-30T23:59:59+00:00"`
    2. Query `activities` for sport in ['bike', 'run', 'swim'] and `started_at` between cutoff_from and cutoff_to — same columns as check_recent():

    Error isolation: wrap each activity processing in try/except (same as check_recent() — E5 fix). Log exception with `logger.exception`.
